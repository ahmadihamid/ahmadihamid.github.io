now I'm trying to write 'bout "moving Linux system to another Hard drive"
this might be useful when we want to upgrade our hard drive to a bigger one, like my case, or u just want to make a backup of your lovely system, so you won't be bothered when you need to customize your system, upgrading and so on,,



cos of Linux flexibility there are plenty tools around just for backing up or cloning purpose,
but today I'll show you tiny-little-powerful "rsync" cud do that for u,


aye,
let the game begin,
--[ plug in your old and new HD
--[ mount your old system partition and the partition that will be our new system-box
--[ then just run this simple command :

rsync -vrpoglHDIt --exclude=/proc -- progress /old/box/path/ /new/box/path/
i.e
rsync -vrpoglHDIt --exclude=/proc -- progress /media/ubuntu/ /mnt/Ubuntu/

ah, I forgot something,
we need to copy our old partition UUID to new one, so won't need to edit fstab, initramfs etc (actually I do not really know what else)
so copying UUID is the simple way,

--[ u can use tune2fs /dev/sdXY -U UUID i.e :

tune2fs /dev/sda3 -U afd46f1f-da52-4b9d-b8a0-997461bb6d5f

yapz,
that's all.
if you have separated /home partition you need run rsync again for that partition,

rsync -aXS --exclude='/*/.gvfs' /old/home/path/ /new/one/

--[ install bootloader or just add new entry or update it (based on your bootloader)
if you use GRUB2 like me,
run grub-install, update-grub, then welcome your lovely system clone.
